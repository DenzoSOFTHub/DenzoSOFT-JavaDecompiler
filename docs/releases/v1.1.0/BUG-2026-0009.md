# BUG-2026-0009: Long comparison operator

**Type: Bug
**Status**: RELEASED
**Severity**: MEDIUM
**Created**: 2026-03-23
**Resolved**: 2026-03-24

## Description
Comparisons involving `long`, `float`, and `double` types produced incorrect expressions. The `lcmp`, `dcmpg`, `dcmpl`, `fcmpg`, `fcmpl` opcodes were not simplified, resulting in expressions like `(a <=> b) > 0` instead of `a > b`.

## Technical Analysis
The JVM uses comparison opcodes (`lcmp`, `dcmpg`, `dcmpl`, `fcmpg`, `fcmpl`) that return -1, 0, or 1, followed by an `if` instruction comparing against 0. The pattern `(a lcmp b) > 0` should be simplified to `a > b`. The `BooleanSimplifier` was not handling these comparison patterns.

### Suggested Approach
In `BooleanSimplifier`, detect the pattern `(a cmp b) OP 0` where `cmp` is a comparison opcode and `OP` is a relational operator, and simplify to `a OP b`.

### Affected Modules
- `src/main/java/it/denzosoft/javadecompiler/service/converter/transform/BooleanSimplifier.java`

### Estimated Effort
LOW

## Solution Implemented
1. Added simplification of `lcmp`/`fcmp`/`dcmp` patterns: `(a <=> b) OP 0` is simplified to `a OP b` (BUG-2026-0009-20260323-1)
2. Added operator flipping for reversed operands (BUG-2026-0009-20260323-2)

### Files Modified
