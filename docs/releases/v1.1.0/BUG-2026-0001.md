# BUG-2026-0001: Nested if-else missing inner branches

**Type: Bug
**Status**: RELEASED
**Severity**: HIGH
**Created**: 2026-03-23
**Resolved**: 2026-03-24

## Description
Deeply nested if-else structures were not correctly reconstructed. Inner branches were missing or duplicated, producing incorrect control flow in the decompiled output.

## Technical Analysis
The CFG-based control flow reconstruction in `StructuredFlowBuilder` did not correctly follow goto chains when detecting merge points for nested if-else structures. When an if-else chain had multiple levels of nesting, the merge point detection would stop at the first goto rather than following the chain to the true merge point, causing inner branches to be lost or merged incorrectly.

### Suggested Approach
Follow goto chains when detecting if-else merge points. Skip past nested conditionals by finding their actual merge points rather than stopping at intermediate goto targets.

### Affected Modules
- `src/main/java/it/denzosoft/javadecompiler/service/converter/cfg/StructuredFlowBuilder.java`

### Estimated Effort
MEDIUM

## Solution Implemented
Two changes in `StructuredFlowBuilder`:
1. Follow goto chains for nested if-else merge detection -- when a goto target itself contains another conditional, follow through to find the true merge point (line ~691)
2. Skip past nested conditionals by finding their merge point, allowing correct reconstruction of arbitrary nesting depth (line ~755)

### Files Modified
- `src/main/java/it/denzosoft/javadecompiler/service/converter/cfg/StructuredFlowBuilder.java` - Added goto chain following in merge point detection (BUG-2026-0001-20260323-1, BUG-2026-0001-20260323-2)
