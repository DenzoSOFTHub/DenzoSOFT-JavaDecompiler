# BUG-2026-0006: Multi-init for loop

**Type: Bug
**Status**: RELEASED
**Severity**: MEDIUM
**Created**: 2026-03-23
**Resolved**: 2026-03-24

## Description
For loops with multiple initializers (e.g., `for (int i = 0, j = 10; ...)`) or multiple update expressions (e.g., `for (...; i++, j--)`) were not correctly reconstructed. Only the first initializer/update was emitted.

## Technical Analysis
The `ForLoopDetector` transform was only looking for a single statement before the while loop as the initializer, and a single trailing statement in the loop body as the update. Multi-init and multi-update patterns produce multiple consecutive statements.

### Suggested Approach
Extend `ForLoopDetector` to collect multiple consecutive assignment/increment statements as for-loop initializers and updates, and extend `JavaSourceWriter` to render them as comma-separated lists.

### Affected Modules
- `src/main/java/it/denzosoft/javadecompiler/service/converter/transform/ForLoopDetector.java`
- `src/main/java/it/denzosoft/javadecompiler/service/writer/JavaSourceWriter.java`

### Estimated Effort
MEDIUM

## Solution Implemented
1. Extended `ForLoopDetector` to detect and collect multiple init and update statements for for-loop patterns (BUG-2026-0006-20260324-1)
2. Extended `JavaSourceWriter` to write multi-init and multi-update for loop parts as comma-separated expressions (BUG-2026-0006-20260324-2)
