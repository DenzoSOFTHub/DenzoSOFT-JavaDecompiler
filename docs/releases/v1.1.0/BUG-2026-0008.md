# BUG-2026-0008: Synchronized block

**Type: Bug
**Status**: RELEASED
**Severity**: MEDIUM
**Created**: 2026-03-23
**Resolved**: 2026-03-24

## Description
Synchronized blocks were not reconstructed from the `monitorenter`/`monitorexit` bytecode pattern. The decompiler showed raw monitor operations instead of `synchronized (obj) { ... }` syntax.

## Technical Analysis
Java `synchronized` blocks compile to:
```
aload <monitor-ref>
dup
monitorenter
... body ...
monitorexit
goto after
catch-handler:
monitorexit   // finally handler for exception path
athrow
```
The decompiler was not recognizing this pattern and was emitting the individual opcodes as statements.

### Suggested Approach
Emit sync markers during bytecode decoding for `monitorenter`/`monitorexit`. In a post-processing pass, reconstruct `synchronized` blocks from the marker pairs.

### Affected Modules
- `src/main/java/it/denzosoft/javadecompiler/service/converter/ClassFileToJavaSyntaxConverter.java`

### Estimated Effort
MEDIUM

## Solution Implemented
1. Emit sync markers during bytecode decoding when `monitorenter`/`monitorexit` instructions are encountered (BUG-2026-0008-20260324-1)
2. Post-processing pass to reconstruct synchronized blocks from monitor marker pairs (BUG-2026-0008-20260324-2, -3)

### Files Modified
- `src/main/java/it/denzosoft/javadecompiler/service/converter/ClassFileToJavaSyntaxConverter.java` - Sync markers for monitorenter/monitorexit, synchronized block reconstruction
