# BUG-2026-0004: Lambda variable types

**Type: Bug
**Status**: RELEASED
**Severity**: MEDIUM
**Created**: 2026-03-23
**Resolved**: 2026-03-24

## Description
Lambda parameter types were incorrectly resolved when the lambda's functional interface method had generic type parameters. The type parsing did not correctly skip `<...>` sections in generic signatures, leading to wrong or missing parameter types.

## Technical Analysis
When parsing method descriptors that include generic signatures (e.g., `<E>(Ljava/util/List<TE;>;)TE;`), the parser was not correctly handling the `<...>` type parameter sections. It would misinterpret characters within the generic bounds as part of the parameter list, producing incorrect types.

### Suggested Approach
Fix the signature parser to correctly skip `<...>` sections by tracking angle bracket nesting depth before parsing the parameter types.

### Affected Modules
- `src/main/java/it/denzosoft/javadecompiler/service/converter/ClassFileToJavaSyntaxConverter.java`

### Estimated Effort
LOW

## Solution Implemented
Added correct generic signature parsing that skips `<...>` type parameter sections by tracking angle bracket nesting depth (BUG-2026-0004-20260324-1).

### Files Modified
- `src/main/java/it/denzosoft/javadecompiler/service/converter/ClassFileToJavaSyntaxConverter.java` - Correctly parse generic signatures by skipping `<...>` sections (line ~2108)
