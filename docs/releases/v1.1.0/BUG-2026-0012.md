# BUG-2026-0012: Enum synthetic members

**Type: Bug
**Status**: RELEASED
**Severity**: HIGH
**Created**: 2026-03-23
**Resolved**: 2026-03-24

## Description
Enum classes showed compiler-generated synthetic members that should be hidden: `$VALUES` array field, `values()` method, `valueOf()` method, `$values()` helper, the `name`/`ordinal` constructor parameters, and the `super(name, ordinal)` call.

## Technical Analysis
The Java compiler generates several synthetic members for enums:
- `private static final EnumType[] $VALUES` field
- `public static EnumType[] values()` method (copies $VALUES)
- `public static EnumType valueOf(String)` method (calls Enum.valueOf)
- `private static EnumType[] $values()` helper (JDK 14+)
- Constructor takes hidden `(String name, int ordinal)` as first 2 parameters
- Constructor body starts with `super(name, ordinal)` call to Enum.<init>
- `<clinit>` initializes enum constants and $VALUES array

### Suggested Approach
Suppress all synthetic enum members in the writer: filter $VALUES field, suppress values/valueOf/$values methods, skip first 2 constructor params, skip super(name, ordinal), suppress entire enum clinit.

### Affected Modules
- `src/main/java/it/denzosoft/javadecompiler/service/writer/JavaSourceWriter.java`

### Estimated Effort
HIGH

## Solution Implemented
Comprehensive enum synthetic member suppression with 9 change tags:
1. Suppress synthetic `$VALUES` field and `$` prefixed fields in inner enums (BUG-2026-0012-20260324-1, -9)
2. Suppress synthetic `values()`, `valueOf()`, `$values()` methods (BUG-2026-0012-20260324-2, -7)
3. Skip first 2 synthetic parameters for enum constructors (BUG-2026-0012-20260324-3)
4. Skip `super(name, ordinal)` call in enum constructors (BUG-2026-0012-20260324-4)
5. Detect super(name, ordinal) pattern (BUG-2026-0012-20260324-5)
6. Suppress entire `<clinit>` for enum classes (BUG-2026-0012-20260324-6)
