# BUG-2026-0011: Assert statement

**Type: Bug
**Status**: RELEASED
**Severity**: MEDIUM
**Created**: 2026-03-23
**Resolved**: 2026-03-24

## Description
Assert statements (`assert condition` and `assert condition : message`) were not reconstructed. The decompiler showed the synthetic `$assertionsDisabled` field and the guard pattern instead of `assert` statements.

## Technical Analysis
Java `assert` compiles to:
```
if (!$assertionsDisabled) {
    if (!condition) {
        throw new AssertionError(message);  // or no-arg
    }
}
```
With a synthetic `static final boolean $assertionsDisabled` field initialized in `<clinit>` via `Class.desiredAssertionStatus()`. The decompiler was showing this raw pattern.

### Suggested Approach
1. Suppress the synthetic `$assertionsDisabled` field from output
2. Skip the `$assertionsDisabled` initialization in `<clinit>`
3. Detect the guard pattern and reconstruct `assert` statements

### Affected Modules
- `src/main/java/it/denzosoft/javadecompiler/service/writer/JavaSourceWriter.java`

### Estimated Effort
MEDIUM

## Solution Implemented
1. Suppress synthetic `$assertionsDisabled` field from field output (BUG-2026-0011-20260323-3)
2. Skip `$assertionsDisabled` init expression in `<clinit>` (BUG-2026-0011-20260323-4, -5)
3. Detect `$assertionsDisabled` initialization expression pattern (BUG-2026-0011-20260323-6)

### Files Modified
- `src/main/java/it/denzosoft/javadecompiler/service/writer/JavaSourceWriter.java` - Suppress $assertionsDisabled field (line ~262), skip clinit init (line ~848, ~856), detect pattern (line ~1862)
