# BUG-2026-0003: Instanceof cast precedence

**Type: Bug
**Status**: RELEASED
**Severity**: MEDIUM
**Created**: 2026-03-23
**Resolved**: 2026-03-24

## Description
Cast expressions used as method receivers or in complex expressions were not parenthesized, producing code that would not compile. For example, `(String) obj.length()` instead of `((String) obj).length()`.

## Technical Analysis
When a cast expression was followed by a method call or field access, the cast needed to be wrapped in parentheses to bind correctly. The writer was emitting the cast without parentheses, causing the method call to bind to the pre-cast expression rather than the cast result.

### Suggested Approach
In `JavaSourceWriter`, detect when a cast expression is used as a method receiver and wrap it in parentheses.

### Affected Modules
- `src/main/java/it/denzosoft/javadecompiler/service/writer/JavaSourceWriter.java`

### Estimated Effort
LOW

## Solution Implemented
Added parenthesization of cast expressions when used as method receivers in `JavaSourceWriter` (BUG-2026-0003-20260323-1). The writer now detects cast expressions in receiver position and wraps them with `()`.

### Files Modified
- `src/main/java/it/denzosoft/javadecompiler/service/writer/JavaSourceWriter.java` - Parenthesize cast expressions used as method receiver (line ~1372)
