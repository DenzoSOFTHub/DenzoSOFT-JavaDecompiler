# LIM-0006: Text blocks

**Type**: Limitation
**Status**: RELEASED
**Severity**: LOW
**Created**: 2026-03-23
**Resolved**: 2026-03-24

## Description
Text blocks (Java 13+) were not distinguished from regular strings in the decompiled output. Multi-line string constants containing newlines were emitted as regular string literals with `\n` escapes instead of text block syntax (`""" ... """`).

## Technical Analysis
Text blocks compile to regular `String` constants in the constant pool. At the bytecode level, there is no difference between a text block and a regular string. The only heuristic available is: if a string literal contains newline characters and the class file version is Java 15+ (59.0+, when text blocks were finalized), it was likely a text block.

### Suggested Approach
Track the class file major version in `JavaSourceWriter`. When emitting a string constant that contains `\n` characters and the major version is >= 59 (Java 15), emit it as a text block instead of a regular string literal.

### Affected Modules
- `src/main/java/it/denzosoft/javadecompiler/service/writer/JavaSourceWriter.java`

### Estimated Effort
LOW

## Solution Implemented
1. Track major version in `JavaSourceWriter` for text block support (LIM-0006-20260324-1, -2)
2. Emit text block syntax for Java 15+ strings containing newlines (LIM-0006-20260324-3)

### Files Modified
