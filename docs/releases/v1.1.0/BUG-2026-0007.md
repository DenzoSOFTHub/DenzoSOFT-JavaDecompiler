# BUG-2026-0007: Labeled break

**Type: Bug
**Status**: RELEASED
**Severity**: MEDIUM
**Created**: 2026-03-23
**Resolved**: 2026-03-24

## Description
Labeled break statements (`break outer;`) were not detected. The `goto` targeting a point beyond the current loop's scope was not recognized as a labeled break, causing incorrect control flow reconstruction.

## Technical Analysis
Labeled breaks in Java compile to `goto` instructions that target a PC beyond the current loop's merge/exit point, jumping to an outer loop's exit. The `StructuredFlowBuilder` did not track outer loop merge points, so it could not distinguish a labeled break (goto to outer loop exit) from a regular branch.

### Suggested Approach
Track outer loop merge points in a stack. When a goto targets beyond the current loop's stopPc but matches an outer loop's merge point, emit a labeled break. Wrap the target loop with a `LabelStatement`.

### Affected Modules
- `src/main/java/it/denzosoft/javadecompiler/service/converter/cfg/StructuredFlowBuilder.java`
- `src/main/java/it/denzosoft/javadecompiler/service/converter/transform/ForLoopDetector.java`

### Estimated Effort
HIGH

## Solution Implemented
Comprehensive labeled break support across 11 change tags:
1. Track outer loop merge points in a stack for labeled break detection (BUG-2026-0007-20260324-1)
2. Push/pop outer loop merge points around loop construction (BUG-2026-0007-20260324-2, -3)
3. Wrap loops with LabelStatement when labeled break targets them (BUG-2026-0007-20260324-4, -5)
4. Handle LabelStatement wrapping in ForLoopDetector (BUG-2026-0007-20260324-6, -7, -8, -9)
5. Don't follow merge beyond outer loop exit (BUG-2026-0007-20260324-10)
6. Skip false merge detection when merge point is beyond stopPc (BUG-2026-0007-20260324-11)

### Files Modified
- `src/main/java/it/denzosoft/javadecompiler/service/converter/cfg/StructuredFlowBuilder.java` - Outer loop merge stack, labeled break detection, label wrapping
- `src/main/java/it/denzosoft/javadecompiler/service/converter/transform/ForLoopDetector.java` - Handle LabelStatement wrapping, preserve labels on converted for-loops
