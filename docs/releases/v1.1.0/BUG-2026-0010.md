# BUG-2026-0010: Constructor this() vs super()

**Type: Bug
**Status**: RELEASED
**Severity**: MEDIUM
**Created**: 2026-03-23
**Resolved**: 2026-03-24

## Description
Constructor chaining with `this()` was not distinguished from `super()`. All `invokespecial <init>` calls in constructors were emitted as `super()`, even when calling another constructor of the same class.

## Technical Analysis
Both `this()` and `super()` compile to `invokespecial <init>`. The difference is the target class: `this()` calls `<init>` on the current class, while `super()` calls `<init>` on the superclass. The decompiler was not checking the target class of the `invokespecial` to distinguish the two cases.

### Suggested Approach
Store the current class internal name during decompilation. When processing an `invokespecial <init>` call, compare the target class with the current class to emit `this()` or `super()`.

### Affected Modules
- `src/main/java/it/denzosoft/javadecompiler/service/converter/ClassFileToJavaSyntaxConverter.java`

### Estimated Effort
LOW

## Solution Implemented
1. Store current class internal name for this() vs super() detection (BUG-2026-0010-20260323-2)
2. Track current class internal name field (BUG-2026-0010-20260323-1)
3. Compare target class of invokespecial `<init>` with current class to distinguish this() from super() (BUG-2026-0010-20260323-3)
