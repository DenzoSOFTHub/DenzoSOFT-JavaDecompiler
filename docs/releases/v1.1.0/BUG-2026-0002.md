# BUG-2026-0002: Array initialization invalid code

**Type: Bug
**Status**: RELEASED
**Severity**: HIGH
**Created**: 2026-03-23
**Resolved**: 2026-03-24

## Description
Array initializer expressions like `new int[] {1, 2, 3}` were not reconstructed from the bytecode pattern of `newarray` followed by a sequence of `dup`, `iconst`, `iastore` instructions. Instead, the decompiler produced invalid code showing individual store operations.

## Technical Analysis
The Java compiler compiles `new int[] {1, 2, 3}` to:
```
newarray int[3]
dup; iconst_0; iconst_1; iastore
dup; iconst_1; iconst_2; iastore
dup; iconst_2; iconst_3; iastore
```
The decompiler was creating a `NewArrayExpression` for the `newarray` but not collecting the subsequent store values into an initializer list.

### Suggested Approach
Detect the `newarray` + sequential `dup/const/store` pattern and collect the stored values as initializer values on the `NewArrayExpression`.

### Affected Modules
- `src/main/java/it/denzosoft/javadecompiler/model/javasyntax/expression/NewArrayExpression.java`
- `src/main/java/it/denzosoft/javadecompiler/service/writer/JavaSourceWriter.java`

### Estimated Effort
MEDIUM

## Solution Implemented
1. Added `initValues` list field to `NewArrayExpression` to store initializer values (BUG-2026-0002-20260323-1, BUG-2026-0002-20260323-2)
2. In `JavaSourceWriter`, emit array initializer syntax `new int[] {1, 2, 3}` when init values are present (BUG-2026-0002-20260323-4)

### Files Modified
- `src/main/java/it/denzosoft/javadecompiler/model/javasyntax/expression/NewArrayExpression.java` - Added initValues field and getter/setter
- `src/main/java/it/denzosoft/javadecompiler/service/writer/JavaSourceWriter.java` - Emit array initializer syntax when init values present
