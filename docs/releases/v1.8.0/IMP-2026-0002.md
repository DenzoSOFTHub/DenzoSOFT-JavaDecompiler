# IMP-2026-0002: Decompilation diagnostics in generated source

**Type**: Improvement
**Status**: RELEASED
**Severity**: MEDIUM
**Created**: 2026-04-20
**Resolved**: 2026-04-20

## Description

When decompilation could not fully reconstruct a class or method, the output previously contained invalid code silently (a placeholder `0` used as a method receiver, a silently-skipped inner class, a linear-scan fallback whose control flow was degraded). The reader had no signal of what was wrong or where.

The user-facing requirement: *"l'utente deve capire dove la decompilazione è fallita ed il tipo di problema che è stato trovato."*

## Technical Analysis

Three categories of silent recovery existed:

1. **Stack underflow**: empty operand stack forced a placeholder expression (`IntegerConstantExpression(0)` or `NullExpression`).
2. **Decoder exceptions**: caught by a try/catch around `decodeOpcode`, emitting a single `/* ERROR */` comment but no aggregate signal.
3. **Pipeline fallback**: CFG build failure → linear-scan fallback, structured-flow failure → linear-scan fallback, inner-class load failure → silently omitted.

### Suggested Approach

Add a per-method diagnostics collector on the converter instance. Funnel every silent-recovery code path through a `recordDiagnostic(note)` helper. Attach the collected notes to `MethodDeclaration`, plus a class-level notes list on `JavaSyntaxResult` for outer-class events (skipped inner class, etc.). In the writer, render the notes as a comment banner at the class top and a comment block before each method body.

## Solution Implemented

- `currentMethodDiagnostics`, `currentDecodePc`, `currentDecodeOpcode` fields on the converter.
- `popOrUnderflowInt` / `popOrUnderflowRef` helpers replace all `stack.isEmpty() ? default : stack.pop()` patterns (~56 call sites) and record `STACK_UNDERFLOW pc=... opcode=...` diagnostics.
- CFG / structured-flow catch blocks record `CFG_BUILD_FAILED` / `STRUCTURED_FLOW_FAILED`.
- Decoder exception handler records `DECODE_ERROR`.
- Inner-class load failure records `INNER_CLASS_SKIPPED` on the outer class.
- `MethodDeclaration.decompilationNotes` + `JavaSyntaxResult.decompilationNotes` carry the lists downstream.
- Writer renders a class-level `WARNING` banner and per-method `// === DECOMPILATION NOTES ===` block.
