# BUG-2026-0050: Exception handler block entry emits `null = e` instead of using caught exception

**Type**: Bug
**Status**: RELEASED
**Severity**: HIGH
**Created**: 2026-04-20
**Resolved**: 2026-04-20

## Description

On entry to a catch handler block, the operand stack holds exactly one value — the caught exception — placed there by the JVM. The decoder did not model this and emitted a STACK_UNDERFLOW placeholder for the subsequent `astore_N` that would have captured the exception into a local. In Spring Boot jars this accounted for hundreds of spurious placeholders.

## Technical Analysis

`BasicBlock` had no "exception handler entry" flag. The CFG builder added handler PCs as block leaders (so handlers started at block boundaries) but nothing propagated the operand-stack seed. When `decodeBasicBlock` ran with an empty stack and hit the handler's opening `astore`, `popOrUnderflowRef` returned a placeholder.

### Suggested Approach

- Add `isExceptionHandler` + `exceptionHandlerType` fields on `BasicBlock`.
- In `ControlFlowGraph.build`, after blocks are constructed, walk `exceptionTable` and mark each handler-entry block.
- In `decodeBasicBlock`, when the block is a handler, push a `LocalVariableExpression("$exception", <handler type>)` onto the stack before decoding starts.

### Affected Modules

- `src/main/java/it/denzosoft/javadecompiler/service/converter/cfg/BasicBlock.java`
- `src/main/java/it/denzosoft/javadecompiler/service/converter/cfg/ControlFlowGraph.java`
- `src/main/java/it/denzosoft/javadecompiler/service/converter/ClassFileToJavaSyntaxConverter.java`

### Estimated Effort

LOW

## Solution Implemented

As described above. `exceptionHandlerType` defaults to `java/lang/Throwable` (covers both typed catch and finally-style catch-all).

### Files Modified

- `src/main/java/it/denzosoft/javadecompiler/service/converter/cfg/BasicBlock.java` — new fields.
- `src/main/java/it/denzosoft/javadecompiler/service/converter/cfg/ControlFlowGraph.java` — flag setting during build.
- `src/main/java/it/denzosoft/javadecompiler/service/converter/ClassFileToJavaSyntaxConverter.java` — stack pre-seed in `decodeBasicBlock`.

### Test Case

- On `contrp.be-springboot-22.2.66.jar`: STACK_UNDERFLOW incidents dropped from 1,425 to 864 (handler-seed fix alone). Combined with BUG-2026-0051, total drops to 0.
