# BUG-2026-0048: package-info classes rendered as `interface package-info {}`

**Type**: Bug
**Status**: RELEASED
**Severity**: MEDIUM
**Created**: 2026-04-20
**Resolved**: 2026-04-20

## Description

Every Java package may have a synthetic class whose binary name is `X/package-info`. The previous output was:

```java
package java.net;

interface package-info {
}
```

`package-info` is not a legal Java identifier (contains `-`), so this fails to compile. Observed in `java/net/package-info.java` and similar.

## Technical Analysis

`writeCompilationUnit` fell through to the generic interface/class writer for any class whose access flags implied interface, regardless of the special name. The correct source representation of a `package-info` class is a `package X;` declaration carrying the class-level annotations.

### Suggested Approach

Detect `package-info` by simple name at the start of `writeCompilationUnit` and emit `[annotations]` + `package X;` only, then return.

### Affected Modules

- `src/main/java/it/denzosoft/javadecompiler/service/writer/JavaSourceWriter.java`

### Estimated Effort

LOW

## Solution Implemented

Early-return branch added to `writeCompilationUnit`.

### Files Modified

- `src/main/java/it/denzosoft/javadecompiler/service/writer/JavaSourceWriter.java`

### Test Case

- Verified on `java.base`: the `<identifier> expected` error at `java/net/package-info.java:4` is gone.
