# BUG-2026-0046: `<T extends A extends B>` emitted for multi-bound type parameters

**Type**: Bug
**Status**: RELEASED
**Severity**: HIGH
**Created**: 2026-04-20
**Resolved**: 2026-04-20

## Description

Generic type parameters with both a class bound and an interface bound rendered as:

```java
public abstract sealed class AbstractLayout<L extends AbstractLayout<L> extends MemoryLayout> ...
```

The repeated `extends` is illegal Java. Observed in `jdk/internal/foreign/layout/AbstractLayout.java`, `AbstractGroupLayout.java`, `ValueLayouts.java`, `java/net/SocketImpl.java`.

## Technical Analysis

In `SignatureParser.parseTypeParameters` the flag tracking "first bound has already been emitted with extends" (`hasClassBound`) was only set inside the interface-bounds loop, never when the class bound was emitted. Consequence: if a non-Object class bound was present and interface bounds followed, the first interface bound saw `hasClassBound == false` and produced a second `extends` instead of `&`.

### Suggested Approach

Rename/reuse the flag (`extendsEmitted`) and set it as soon as either the class bound or the first interface bound is appended.

### Affected Modules

- `src/main/java/it/denzosoft/javadecompiler/util/SignatureParser.java`

### Estimated Effort

LOW

## Solution Implemented

Single-line logic fix in `parseTypeParameters`.

### Files Modified

- `src/main/java/it/denzosoft/javadecompiler/util/SignatureParser.java`

### Test Case

- Verified on `java.base`: the 4 `> expected` + `illegal start of type` errors tied to multi-bound generics are gone.
