# IMP-2026-0004: CLI output options and line number alignment fix

**Type**: Improvement
**Status**: CLOSED
**Severity**: MEDIUM
**Created**: 2026-03-26
**Resolved**: 2026-03-26

## Description
Add CLI flags to control decompiler output: line number alignment (default), compact mode, bytecode metadata display, and JNI native method info. Fix Printer line tracking so aligned mode correctly preserves original source line numbers.

## Technical Analysis
The Printer implementations did not increment `currentLine` in `endLine()`, causing gap calculations to produce incorrect line alignment. The decompiler always emitted JNI comments for native methods with no way to disable them.

### Suggested Approach
- Fix Printer `endLine()` to advance `currentLine`
- Make line alignment the default, add `--compact` to disable
- Add `--show-bytecode` and `--show-native-info` as opt-in flags
- Pass flags through the Message configuration map to the writer

### Affected Modules
- `Main.java` - CLI flag parsing, StringPrinter
- `BatchDecompiler.java` - StringCollector, constructor
- `JavaSourceWriter.java` - Conditional JNI comments, bytecode metadata
- `JavaSyntaxResult.java` - MethodDeclaration bytecode fields
- `ClassFileToJavaSyntaxConverter.java` - Populate bytecode metadata

### Estimated Effort
MEDIUM

## Solution Implemented
1. **Printer fix**: `endLine()` increments `currentLine`; `startLine()` gap formula changed from `lineNumber > currentLine + 1` to `lineNumber > currentLine`; `currentLine` initialized to 1; never set backwards.
2. **Default = aligned**: Line number preservation is the default output mode.
3. **`--compact`**: Disables line alignment for dense readable output.
4. **`--show-bytecode`**: Emits `// Bytecode: N bytes, max_stack=X, max_locals=Y` comment at the start of each method body.
5. **`--show-native-info`**: Emits `// JNI: Java_pkg_Class_method | params: (...)` on native method declarations. Without this flag, native methods are shown clean.
6. **Configuration propagation**: Flags passed via `Map<String, Object> configuration` through `DenzoDecompiler.decompile()` to `JavaSourceWriter` via Message headers.
7. **MethodDeclaration extended** with `bytecodeLength`, `maxStack`, `maxLocals` fields populated from CodeAttribute.
