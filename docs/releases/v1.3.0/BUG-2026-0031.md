# BUG-2026-0031: Generic unchecked cast T lost in output

**Type**: Bug
**Status**: RESOLVED
**Severity**: LOW
**Created**: 2026-03-25
**Resolved**: 2026-03-25

## Description

`return (T) obj` showed as `return obj` - the generic cast was suppressed.

## Solution Implemented

1. In the converter, after decompiling the method body, check if the method's generic signature
   has a type variable return type. If so, wrap return expressions in CastExpression with GenericType.
2. In the writer, never suppress casts to GenericType (type variables like T).

### Files Modified
- `service/converter/ClassFileToJavaSyntaxConverter.java` - addGenericReturnCasts post-processing
- `service/writer/JavaSourceWriter.java` - Don't suppress GenericType casts

### Test Case
- Test class: `TestAuditV2`
- Test method: `unsafeCast(Object)`
- Verifies: Generic cast `(T)` is preserved in output
