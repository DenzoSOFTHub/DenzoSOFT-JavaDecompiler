# BUG-2026-0026: while(true) with break/return not reconstructed

**Type**: Bug
**Status**: RESOLVED
**Severity**: CRITICAL
**Created**: 2026-03-25
**Resolved**: 2026-03-25

## Description

`while (true) { if (cond1) return x; if (cond2) return y; i++; }` showed as flat if-statements
without loop wrapper.

## Solution Implemented

Added pre-scan detection of unconditional backward gotos that target non-conditional blocks
(or conditional blocks that don't form while-condition loops). When such a pattern is found,
the body is wrapped in `WhileStatement(true, body)`.

### Files Modified
- `service/converter/cfg/StructuredFlowBuilder.java` - Added whileTrueHeaders detection and while(true) reconstruction

### Test Case
- Test class: `TestAuditV2`
- Test method: `findIndex(int[], int)`
- Verifies: while(true) loop with if-return pattern is reconstructed
