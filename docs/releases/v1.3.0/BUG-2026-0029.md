# BUG-2026-0029: Anonymous class displayed as new 1() - invalid Java name

**Type**: Bug
**Status**: RESOLVED
**Severity**: HIGH
**Created**: 2026-03-25
**Resolved**: 2026-03-25

## Description

Anonymous inner class `TestAuditV2$1` displayed as `new 1(this)` instead of using the
interface/superclass name.

## Solution Implemented

1. Built a map of anonymous class internal names to their interface/superclass display names
2. When writing a NewExpression for an anonymous class, use the interface/superclass name
   and suppress the outer `this` argument
3. Suppress anonymous inner class declarations in the output (they can't be referenced by name)

### Files Modified
- `service/writer/JavaSourceWriter.java` - Anonymous class display name map, NewExpression fix, inner class suppression

### Test Case
- Test class: `TestAuditV2`
- Test method: `getComparator()`
- Verifies: Shows `new Comparator()` instead of `new 1(this)`
