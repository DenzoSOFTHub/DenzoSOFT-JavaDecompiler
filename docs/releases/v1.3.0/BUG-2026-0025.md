# BUG-2026-0025: 4+ chained && operators: result lost, returns false

**Type**: Bug
**Status**: RESOLVED

## Description

`s != null && s.length() >= min && s.length() <= max && !s.trim().isEmpty()` produced `return false`.

## Solution Implemented

Same root cause as BUG-2026-0024. The ternary merge block detection was fixed to handle
RETURN blocks with a single ReturnStatement.
