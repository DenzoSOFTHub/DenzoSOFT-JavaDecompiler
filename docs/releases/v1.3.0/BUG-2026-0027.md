# BUG-2026-0027: Multi-field equals comparison returns false

**Type**: Bug
**Status**: RESOLVED
**Severity**: CRITICAL
**Created**: 2026-03-25
**Resolved**: 2026-03-25

## Description

`id == t.id && Double.compare(score, t.score) == 0 && Objects.equals(name, t.name)` produced `return false`.

## Solution Implemented

Two fixes:
1. Same ternary merge fix as BUG-0025 (the && chain + ternary detection)
2. Fixed BooleanSimplifier to not treat int-returning method calls as boolean.
   `looksBoolean()` now returns false for MethodInvocationExpression and
   StaticMethodInvocationExpression when their return type is PrimitiveType.INT.
   This prevents `Double.compare() == 0` from being simplified to `!Double.compare()`.

### Files Modified
- `service/converter/cfg/StructuredFlowBuilder.java` - Ternary merge fix
- `service/converter/transform/BooleanSimplifier.java` - Don't treat int-returning methods as boolean

### Test Case
- Test class: `TestAuditV2`
- Test method: `equals(Object)`
- Verifies: Multi-field equals with Double.compare produces correct output
