# BUG-2026-0024: Nested ternary beyond 2 levels returns only last value

**Type**: Bug
**Status**: RESOLVED
**Severity**: CRITICAL
**Created**: 2026-03-25
**Resolved**: 2026-03-25

## Description

`x > 0 ? "positive" : x < 0 ? "negative" : "zero"` returns just `"zero"`.

The ternary detector in StructuredFlowBuilder handles 2-level nesting but NOT 3+ levels.

## Technical Analysis

The `canFormTernary` method used `canFormTernarySimple` for nested branches, which only checked
one level deep. For 3+ levels, the recursive check was needed.

Additionally, the merge block detection for ternaries failed when the RETURN merge block had
a non-empty statements list (due to stack seeding from predecessors).

## Solution Implemented

1. Changed `canFormTernary` to recursively call itself instead of `canFormTernarySimple`
2. Fixed merge block detection to treat RETURN blocks with a single ReturnStatement as pure returns

### Files Modified
- `service/converter/cfg/StructuredFlowBuilder.java` - Recursive canFormTernary, pureReturn fix
