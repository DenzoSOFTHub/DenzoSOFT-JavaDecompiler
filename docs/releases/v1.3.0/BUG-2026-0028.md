# BUG-2026-0028: Lambda predicate body shows false instead of actual expression

**Type**: Bug
**Status**: RESOLVED
**Severity**: HIGH
**Created**: 2026-03-25
**Resolved**: 2026-03-25

## Description

`s -> !s.isEmpty()` showed as `s -> false`.

## Solution Implemented

Same root cause as BUG-0025: the ternary merge block detection in StructuredFlowBuilder
failed for the lambda's synthetic method body because the RETURN merge block had a non-empty
statements list. The pureReturn fix resolves this.

### Files Modified
- `service/converter/cfg/StructuredFlowBuilder.java` - Extended pureReturn check
