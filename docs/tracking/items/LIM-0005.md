# LIM-0005: Pattern matching for switch (Java 21+) not detected

**Type**: Limitation
**Status**: OPEN
**Severity**: LOW
**Created**: 2026-03-23

## Description
Pattern matching for switch (Java 21+, JEP 441) generates bytecode using `invokedynamic` with `SwitchBootstraps.typeSwitch` or `SwitchBootstraps.enumSwitch`. The decompiler does not reconstruct the high-level pattern switch syntax and instead shows the compiled form (index-based switch), which is functionally correct but not idiomatic.

The bytecode pattern is:
```
invokedynamic typeSwitch(Object) -> int   // maps object to case index
tableswitch/lookupswitch on the index     // dispatch to case body
```

## Technical Analysis
Pattern matching for switch uses `java.lang.runtime.SwitchBootstraps` bootstrap methods introduced in Java 21. The pattern is:

1. An `invokedynamic` call to `typeSwitch` or `enumSwitch` that takes the switch selector object and returns an integer case index
2. The bootstrap method arguments encode the type patterns (class references) and constant patterns
3. A `tableswitch` or `lookupswitch` on the returned index dispatches to case bodies
4. Guard expressions (when clauses) generate additional conditional branches within case bodies

Full reconstruction requires:
- Detecting `SwitchBootstraps.typeSwitch` / `enumSwitch` invokedynamic calls
- Parsing the bootstrap method arguments to extract type pattern classes and constants
- Mapping the integer case indices back to the original pattern types
- Reconstructing `when` guard clauses from conditional branches
- Handling nested patterns and record patterns (Java 21+)

The bytecode encoding has evolved across JDK preview versions (17-20) before being finalized in Java 21, adding complexity.

### Suggested Approach
For a future version:
1. In `ClassFileToJavaSyntaxConverter`, detect `invokedynamic` calls targeting `SwitchBootstraps`
2. Extract bootstrap method arguments to map case indices to type patterns
3. In `StringSwitchReconstructor` (or a new `PatternSwitchReconstructor`), merge the invokedynamic + tableswitch into a pattern switch AST node
4. In `JavaSourceWriter`, render pattern switch cases with type patterns

### Affected Modules
- `src/main/java/it/denzosoft/javadecompiler/service/converter/ClassFileToJavaSyntaxConverter.java` - invokedynamic detection
- `src/main/java/it/denzosoft/javadecompiler/service/converter/transform/StringSwitchReconstructor.java` - Pattern similar to string switch
- `src/main/java/it/denzosoft/javadecompiler/service/writer/JavaSourceWriter.java` - Pattern switch rendering

### Estimated Effort
HIGH - Requires bootstrap method argument parsing, type pattern mapping, when-clause reconstruction, and a new AST transform pass.

## Reason for Remaining Open
Pattern matching for switch (Java 21+) uses `SwitchBootstraps.typeSwitch` invokedynamic which varies between JDK preview versions. Full reconstruction requires tracking the bootstrap method arguments to map type patterns to case indices and reconstructing guard clauses. The decompiler shows the compiled form (index-based switch) which is functionally correct. This is a cosmetic limitation for modern Java 21+ code.
