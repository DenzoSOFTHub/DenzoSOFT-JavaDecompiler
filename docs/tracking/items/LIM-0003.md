# LIM-0003: @Override not reconstructable

**Type**: Limitation
**Status**: PERMANENT
**Severity**: LOW
**Created**: 2026-03-23

## Description
The `@Override` annotation cannot be reconstructed by the decompiler because it has `RetentionPolicy.SOURCE` and is therefore not present in class files.

## Technical Analysis
The `@Override` annotation is defined with `@Retention(RetentionPolicy.SOURCE)` in the JDK. Source-retention annotations are discarded by the Java compiler during compilation and are not written to the class file. Since the class file contains no record of `@Override`, no decompiler can reconstruct it from bytecode alone.

Theoretically, `@Override` could be inferred by checking whether a method overrides a superclass method or implements an interface method, but this would require:
1. Loading the superclass/interface class files
2. Resolving the full type hierarchy
3. Matching method signatures accounting for generics and bridge methods

This is beyond the scope of a single-class decompiler.

### Affected Modules
None -- this is an inherent limitation of class file format.

### Estimated Effort
N/A -- Cannot be fixed without full type hierarchy resolution.

## Reason for Permanent Limitation
`@Override` has `RetentionPolicy.SOURCE` per JLS/JVM spec. Source-retention annotations are not stored in class files. This is a fundamental limitation of bytecode decompilation, not a decompiler bug.
